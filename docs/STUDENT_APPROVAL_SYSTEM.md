# نظام قبول طلبات الطلاب - Student Approval System

## نظرة عامة

تم تطوير نظام متكامل لإدارة قبول طلبات الانضمام للمنصة التعليمية، حيث يتم تعليق حسابات الطلاب الجدد تلقائياً في انتظار تفعيل الإدارة.

## الميزات الرئيسية

### 1. تعليق الحسابات تلقائياً
- عند إنشاء حساب طالب جديد، يتم ضبط الحالة تلقائياً على `pending` (قيد المراجعة)
- لا يمكن للطالب استخدام المنصة إطلاقاً حتى يتم تفعيل حسابه
- يتم توجيه الطالب تلقائياً لصفحة انتظار التفعيل

### 2. صفحة انتظار التفعيل (`/pending-approval`)
- تعرض للطالب حالة طلبه بوضوح
- توضح خطوات العملية: إنشاء الحساب → المراجعة الإدارية → التفعيل
- تمكن الطالب من مراسلة الإدارة للاستعجال أو الاستفسار
- تحديث تلقائي للحالة - عند التفعيل يتم توجيه الطالب فوراً للوحة التحكم

### 3. لوحة إدارة قبول الطلاب (`/admin/student-approvals`)
- عرض جميع طلبات الانضمام المعلقة
- فلترة حسب الحالة (قيد المراجعة، المقبولون، المرفوضون)
- بحث متقدم بالاسم، البريد، أو الهاتف
- إحصائيات شاملة:
  - عدد الطلاب المعلقين
  - عدد الطلاب المفعّلين
  - عدد المرفوضين

### 4. إجراءات القبول والرفض

#### قبول طالب:
- زر "قبول" يفتح نافذة تأكيد
- عند التفعيل:
  - يتم تحديث الحالة إلى `approved`
  - إضافة الطالب للفصل الدراسي المحدد
  - إرسال إشعار للطالب بالتفعيل
  - تحديث `localStorage` تلقائياً
  - توجيه الطالب للوحة التحكم الخاصة به

#### رفض طالب:
- زر "رفض" يفتح نافذة تطلب سبب الرفض
- يتم تسجيل سبب الرفض في قاعدة البيانات
- إرسال إشعار للطالب يوضح سبب الرفض
- لا يمكن للطالب المرفوض استخدام المنصة

### 5. التكامل مع نظام تسجيل الدخول
- عند تسجيل الدخول، يتم التحقق من حالة الطالب
- إذا كانت الحالة `pending` أو غير محددة → توجيه لصفحة الانتظار
- إذا كانت الحالة `approved` → توجيه للوحة التحكم
- إذا كانت الحالة `rejected` → يبقى في صفحة الانتظار مع عرض سبب الرفض

## سير العمل

### للطالب:
1. ينشئ الطالب حسابه الجديد
2. يتم ضبط الحالة تلقائياً على `pending`
3. يتم توجيهه لصفحة انتظار التفعيل
4. لا يمكنه الوصول لأي جزء من المنصة
5. يمكنه مراسلة الإدارة للاستفسار
6. عند التفعيل، يتم توجيهه تلقائياً للوحة التحكم

### للإدارة:
1. الدخول للوحة التحكم الرئيسية
2. الضغط على "قبول الطلاب" أو الذهاب لـ `/admin/student-approvals`
3. مراجعة الطلبات المعلقة
4. قبول أو رفض كل طلب مع تحديد السبب عند الرفض
5. يتم إشعار الطالب تلقائياً

## التحسينات المستقبلية المقترحة

1. **إشعارات بالبريد الإلكتروني**: إرسال إيميل عند القبول/الرفض
2. **تفعيل جماعي**: تحديد عدة طلاب وتفعيلهم دفعة واحدة
3. **تصفية متقدمة**: حسب المرحلة الدراسية، الفصل، تاريخ التقديم
4. **تقارير**: تصدير تقارير عن الطلبات الشهرية
5. **انتهاء الصلاحية**: رفض تلقائي للطلبات القديمة
6. **مرفقات**: إمكانية رفع مستندات ثبوتية عند التسجيل

## الأمان والصلاحيات

- فقط المستخدمين بصلاحية `admin` أو `super_admin` يمكنهم الوصول لصفحة إدارة القبول
- الطلاب المعلقين لا يمكنهم الوصول لأي صفحة кроме `/pending-approval`
- جميع الإجراءات مسجلة في سجل النشاطات

## الاختبار

### سيناريوهات الاختبار:
1. ✅ إنشاء حساب طالب جديد → الحالة `pending`
2. ✅ محاولة دخول طالب معلق → توجيه لصفحة الانتظار
3. ✅ قبول طالب من الإدارة → إشعار + توجيه للوحة التحكم
4. ✅ رفض طالب مع سبب → إشعار بالسبب
5. ✅ طالب مرفوض لا يمكنه استخدام المنصة
6. ✅ تحديث تلقائي للحالة في الوقت الفعلي

- سجل النشاطات لمراجعة جميع الإجراءات
