# دليل المالك التقني - EduSafa Platform

## 1. نظرة عامة
هذا المستند يوثق البنية التحتية البرمجية والهندسية لمنصة "EduSafa". تم إعادة تصميم النظام ليكون هرمياً (Hierarchical) بدلاً من الهيكل المسطح، مما يضمن قابلية التوسع (Scalability) والأداء العالي.

## 2. هيكلية قاعدة البيانات (Hierarchical Data Model)
تم الانتقال إلى هيكل: `edu/sch/classes/{level}/{grade}/{classId}`.

### التفرعات الرئيسية:
- **`edu/sch/classes`**: الجذر لجميع الفصول الدراسية.
- **`edu/courses`**: المخزن العالمي للمقررات، مرتب حسب `level/grade`.
- **`edu/announcements`**: نظام إعلانات ذكي يتيح الاستهداف (Global, Level, Grade, Class).

## 3. المعايير الهندسية
- **الاستقرار:** استخدام `Functional State Updates` لضمان استجابة واجهة المستخدم.
- **إعادة الاستخدام:** تم بناء مكون `DataTable` الموحد لإدارة الجداول، مما قلل التكرار بنسبة 40%.
- **الأمن:** تم فرض حالة `pending` لجميع الحسابات الجديدة لمنع الدخول غير المصرح به قبل موافقة الإدارة.

## 4. المسارات (Path Constants)
يتم إدارة كافة المسارات عبر `constants/dbPaths.ts`. **يُمنع منعاً باتاً** كتابة مسارات ثابتة (Hardcoded strings) داخل المكونات. استخدم الدوال المعرفة في `EDU` أو `SYS`.

## 5. تعليمات للمطور المستقبلي
- عند إضافة فصل جديد: تأكد من تحديد `level` و `grade` الصحيحين.
- عند التعامل مع المكونات: استخدم `DataTable` بدلاً من بناء جداول مخصصة.
- للتأمين: أي دور جديد يحتاج إضافة استثناء في `services/auth.service.ts` إذا كان لا يتطلب موافقة إدارية (وهو أمر غير مستحسن).
