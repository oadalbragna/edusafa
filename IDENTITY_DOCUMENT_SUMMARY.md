# ✅ نظام رفع مستندات الهوية - ملخص التنفيذ

## 📋 ما تم إنجازه

تم إنشاء نظام كامل لرفع مستندات إثبات الهوية (بطاقة هوية، جواز سفر، إلخ) عبر جسر تيليغرام بنجاح.

## 📁 الملفات المُنشأة

### 1. **مكون رفع مستندات الهوية**
```
components/parent/IdentityDocumentUpload.tsx
```
- مكون تفاعلي لرفع المستندات
- دعم 7 أنواع مختلفة من المستندات
- واجهة بطاقات مرية لاختيار نوع المستند
- معاينة الصور قبل الرفع
- سحب وإفلات الملفات
- التحقق من نوع وحجم الملف (حد أقصى 10 ميجابايت)
- شريط تقدم الرفع
- الرفع عبر **جسر تيليغرام**

### 2. **خدمة إدارة مستندات الهوية**
```
services/identityDocument.service.ts
```
خدمة Firebase كاملة لإدارة المستندات:
- `saveParentIdentityDocument()` - حفظ مستند جديد
- `getParentIdentityDocuments()` - جلب جميع المستندات
- `updateIdentityDocumentStatus()` - تحديث الحالة (موافقة/رفض)
- `getLatestApprovedIdentityDocument()` - جلب آخر مستند موافق عليه
- `hasPendingIdentityDocument()` - التحقق من وجود مستند قيد المراجعة

**هيكل التخزين في Firebase:**
```
sys/users/{parentUid}/identityDocuments/
  └─ {documentId}
      ├─ id: string
      ├─ documentType: string (id_card | passport | family_book | ...)
      ├─ fileUrl: string (رابط Telegram Bridge)
      ├─ fileName: string
      ├─ fileId: string (معرف الملف في Telegram)
      ├─ shortId: string
      ├─ uploadedAt: string
      └─ status: 'pending' | 'approved' | 'rejected'
```

### 3. **صفحة مراجعة المستندات (للإدارة)**
```
pages/Parent/IdentityDocumentReviewPage.tsx
```
صفحة كاملة للإدارة لمراجعة المستندات:
- عرض بيانات ولي الأمر

- إضافة ملاحظات المراجعة
- الموافقة أو رفض المستند
- نموذج تأكيد قبل المعالجة

**المسار:**
```
/admin/identity-documents/review/:parentUid/:documentId
```

## 🔄 سير العمل

```
ولي الأمر:
1. يدخل كود دعوة الطالب
2. ينتظر موافقة الطالب
3. بعد الموافقة، يظهر قسم رفع مستند الهوية
4. يختار نوع المستند
5. يرفع الملف (صورة أو PDF)
6. يتم الرفع عبر جسر تيليغرام
7. يُحفظ المستند في: sys/users/{parentUid}/identityDocuments/
8. الحالة: 'pending'

الإدارة:
1. تتصفح المستندات المعلقة
2. تفتح صفحة المراجعة
3. تراجع بيانات ولي الأمر والمستند
4. توافق أو ترفض مع إضافة ملاحظات
5. يتم تحديث الحالة إلى: 'approved' أو 'rejected'
```

## 🚀 كيفية الاستخدام

### ولي الأمر يرفع مستند هوية:

1. توجه إلى `/parent-accept`
2. أدخل كود دعوة الطالب وانتظر موافقة الطالب
3. بعد موافقة الطالب، يظهر قسم "رفع مستند إثبات الهوية"
4. اختر نوع المستند من البطاقات
5. اضغط على منطقة الرفع أو اسحب الملف
6. راجع المعاينة
7. اضغط "رفع المستند عبر تيليغرام"
8. انتظر حتى يكتمل الرفع

### الإدارة تراجع مستند هوية:

1. توجه إلى `/admin/identity-documents/review/{parentUid}/{documentId}`
2. راجع بيانات ولي الأمر
3. شاهد المستند المرفوع
4. أضف ملاحظاتك (اختياري)
5. اضغط "الموافقة" أو "رفض"
6. أكد العملية

## 📝 ملاحظات هامة

1. **جسر تيليغرام**: جميع الملفات تُرفع عبر جسر تيليغرام الموجود بالفعل في المشروع
2. **الروابط الآمنة**: يتم استخدام `TelegramService.uploadFile()` الذي يعيد رابطاً آمناً
3. **التحقق**: يتم التحقق من نوع الملف (صور/PDF فقط) وحجمه (حد أقصى 10MB)
4. **التخزين**: المستندات تُحفظ في `sys/users/{parentUid}/identityDocuments/` بشكل منفصل عن طلبات الربط
5. **المراجعة**: مستندات الهوية تراجعها الإدارة مباشرة (لا تحتاج مراجعة الطالب)
